# Vibe Stack — Project Context

## Tech Stack
- Framework: Next.js 16.2.1 (App Router) with React 19
- Language: TypeScript (strict mode)
- Styling: Tailwind CSS 4 + shadcn/ui
- Database: Supabase (Postgres + RLS)
- Auth: Supabase SSR + Custom JWT Logic (untuk hybrid web/mobile)
- Validation: Zod (via src/lib/validations.ts)
- Payments: Midtrans (Indonesian local gateway)
- Email: Nodemailer (SMTP based)
- State Management: Zustand + Tanstack Query v5
- Mobile Support: Capacitor 8.3.0
- Icons: lucide-react

## Architecture Rules
1. I18n Routing: All main pages must be located under the route group src/app/[lang]. Use the dictionary loader for UI text.
2. Server-First: Default to Server Components. Use ‘use client’ only for leaf components or those requiring interactivity.
3. Secure API Routes: All API routes in src/app/api must be wrapped with the secureRoute middleware from src/lib/api-middleware.ts for JWT, Role, and Rate Limiting checks.
4. Data Fetching:
   - Server: Use createClient() from @/lib/server.
   - Client: Must use Tanstack Query; do not use manual fetch via useEffect.
5. No getSession: Always use supabase.auth.getUser() on the server side for security reasons.
6. Mobile Friendly: Ensure responsive styling and avoid using browser-only APIs at the root level to prevent breakage when building via Capacitor.

## Folder Structure
```
src/
├── app/                  # Next.js App Router
│   ├── [lang]/           # I18n protected & public pages
│   │   ├── (public)/     # Landing, public info
│   │   ├── admin/        # Admin dashboard & management
│   │   ├── portal/       # User portal
│   │   └── auth/         # Login, register, forgot-password
│   └── api/              # Route handlers (secured via api-middleware)
├── components/           # Reusable components
│   ├── admin/            # Admin-specific components
│   ├── portal/           # Portal-specific components
│   ├── layouts/          # Persistent layouts (Sidebar, Navbar)
│   └── ui/               # shadcn/ui primitives
├── lib/                  # Core logic & utilities
│   ├── server.ts         # Supabase Server Client factory
│   ├── client.ts         # Supabase Browser Client factory
│   ├── api-middleware.ts # API Security (JWT + Rate Limit)
│   ├── validations.ts    # Centralized Zod schemas
│   └── utils.ts          # Tailwind merge & general helpers
├── dictionaries/         # I18n JSON translations
├── types/                # TypeScript interfaces & DB types
└── hooks/                # Custom React hooks (Query & Local state)

```

## Naming Conventions
- Files (Components): PascalCase (e.g., UserTable.tsx, DashboardContent.tsx)
- Files (Routes/Folders): kebab-case (e.g., audit-trail/page.tsx)
- Server Actions: camelCase verbs (e.g., submitPayment, updateUserProfile)
- Database: snake_case (e.g., financial_records, audit_logs)
- Environment: Use the NEXT_PUBLIC_ prefix only for items that are safe for client access.
